The unknown flag: --display_name error is a known bug in some Stripe CLI versions (CLI rejects valid flags like --display_name), but you can work around it by using plain curl or by creating the shipping rates directly in your backend code instead of via the CLI right now.

✅ Option: Create shipping rates in code instead of CLI
In your backend (e.g., server.ts or wherever you call stripe), create a small setup script once.

ts
import { Stripe } from "stripe";

const stripe = new Stripe(process.env.STRIPE_API_KEY, {
  apiVersion: "2025-04-30",
});

async function createShippingRates() {
  const free = await stripe.shippingRates.create({
    display_name: "Envío estándar gratuito (600 MXN en adelante)",
    type: "fixed_amount",
    fixed_amount: {
      amount: 0,
      currency: "MXN",
    },
  });
  console.log("FREE SHIPPING RATE ID:", free.id);

  const paid = await stripe.shippingRates.create({
    type: "fixed_amount",
    display_name: "Envío estándar (MXN)",
    fixed_amount: {
      amount: 1500,
      currency: "MXN",
    },
  });
  console.log("PAID SHIPPING RATE ID:", paid.id);

  return { free, paid };
}
Run this once (e.g., in a little setup.ts file or during app startup). It will print:

free.id → your shr_free (for ≥ 600 MXN).

paid.id → your shr_paid (for < 600 MXN).
